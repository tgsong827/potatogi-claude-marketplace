# Dev Tools Doctor

머신에 필요한 개발 도구들을 진단하고 설치/설정까지 수행한다.
`tools.toml`에 정의된 도구 목록을 읽어 체크 → 설치 → 설정 워크플로우를 진행한다.

## 1단계: 설정 파일 로드

이 스킬 디렉토리의 `tools.toml`을 Read로 읽어 전체 도구 목록을 파악한다.

`tools.toml`은 카테고리별로 도구를 정의한다. 각 도구는 다음 필드를 가진다:
- `name` (필수): 표시 이름
- `check` (필수): 설치 여부 확인 명령 (exit code 0이면 설치됨, 출력에서 버전 추출)
- `install` (필수): 설치 명령
- `setup` (선택): 설치 후 추가 설정
  - `commands`: 실행할 명령 배열
  - `description`: 설정 내용 설명

## 2단계: 진단 (Doctor)

각 도구의 `check` 명령을 Bash로 실행하여 설치 여부와 버전을 확인한다.
출력의 첫 줄에서 버전 번호를 추출한다. 실패 시(exit code != 0) "미설치"로 표시한다.

카테고리별로 결과를 테이블로 출력한다:

```
## 진단 결과

### cli
| 도구 | 상태 | 버전 |
|------|------|------|
| GitHub CLI | :white_check_mark: | 2.50.0 |
| AWS CLI | :x: | 미설치 |

### runtime
| 도구 | 상태 | 버전 |
|------|------|------|
| Node.js | :white_check_mark: | v22.1.0 |
| Bun | :white_check_mark: | 1.1.0 |
```

## 3단계: 설치 대상 확인

미설치 도구가 없으면 "모든 도구가 설치되어 있습니다!" 출력 후 종료한다.

미설치 도구가 있으면 `AskUserQuestion`으로 선택지를 제공한다:
- **header**: "도구 설치"
- **question**: "다음 도구가 설치되지 않았습니다:\n\n- {미설치 도구 목록}\n\n어떻게 진행할까요?"
- **옵션**:
  - "전체 설치" — 미설치 도구 모두 설치
  - "하나씩 선택" — 각 도구마다 설치 여부를 개별 확인
  - "건너뛰기" — 설치 없이 종료

## 4단계: 설치 실행

선택된 도구의 `install` 명령을 순서대로 Bash로 실행한다.

"하나씩 선택"인 경우, 각 도구마다 `AskUserQuestion`으로 설치 여부를 확인한다:
- **header**: "{도구 이름}"
- **question**: "`{install 명령}`을 실행하여 {도구 이름}을 설치할까요?"
- **옵션**: "설치" / "건너뛰기"

각 설치 후 `check` 명령으로 성공 여부를 확인한다. 실패 시 에러 내용을 출력하되 다음 도구로 계속 진행한다.

## 5단계: 추가 설정

설치된 도구 중 `setup`이 정의된 것이 있으면, 각각에 대해:

1. 설정의 `description`과 실행할 `commands`를 보여준다
2. `AskUserQuestion`으로 확인한다:
   - **header**: "{도구 이름} 설정"
   - **question**: "{description}\n\n다음 명령을 실행합니다:\n```\n{commands 목록}\n```\n\n적용할까요?"
   - **옵션**: "적용" / "건너뛰기"
3. "적용" 시 `commands`를 순서대로 Bash로 실행한다

`setup`이 정의된 도구가 없거나 모두 건너뛰면 이 단계를 생략한다.

## 6단계: 최종 결과

설치/설정 작업의 요약을 테이블로 출력한다:

```
## 완료 요약

| 도구 | 설치 | 설정 |
|------|------|------|
| GitHub CLI | :white_check_mark: 설치 완료 | :white_check_mark: 인증 설정 완료 |
| Docker | :x: 설치 실패 | - |
| Bun | :white_check_mark: 이미 설치됨 | 건너뜀 |
```
